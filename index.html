FINAL EXAM RUBRIC

Production-Grade CI/CD System (Total: 10.0 points)

1. Infrastructure Provisioning & Environment Readiness (2.0 points)
Score Description
0.0 No functional cloud environment; system not reachable via public domain.
1.0 Manual provisioning on a real cloud server; domain and HTTPS correctly configured; environment stable but mostly manual.
1.5 Partial automation (Terraform or Ansible or structured scripts); environment reproducible with minor manual steps.
2.0 Full Infrastructure as Code using Terraform and configuration management; idempotent setup; secure and production-ready environment with clear evidence.
2. Architecture & Deployment Model (Tier-Based Execution Quality) (2.5 points)
Score Description
0.0 Deployment fails or system is not operational.
1.0 Tier 1 – Single-server non-containerized deployment using systemd or PM2; operational but limited scalability.
1.5 Tier 2 – Single-server containerized deployment (Docker Compose) with correct service separation and persistent storage.
2.0 Tier 3 – Multi-server architecture with centralized load balancing (managed cloud load balancer preferred); horizontal scaling demonstrated.
2.5 Tier 4 or Tier 5 – Docker Swarm or Kubernetes orchestration with service replication, resilience, and scalable architecture.
3. CI/CD Pipeline Implementation (CI + CD + DevSecOps) (2.5 points)
Score Description
0.0 No working CI/CD pipeline.
1.0 Basic automated pipeline (build → deploy); limited checks and weak versioning.
1.5 Functional CI pipeline (lint, build, caching) and automated CD; security integration minimal or incomplete.
2.0 Complete CI/CD pipeline with versioned artifacts or images, automated deployment, and security scanning integrated.
2.5 Production-grade CI/CD with enforced security fail gates, clean versioning strategy, and controlled deployment behaviour.
4. Monitoring, Observability & Runtime Awareness (1.0 point)
Score Description
0.0 No monitoring system, or monitoring installed but not functioning.
0.5 Monitoring stack operational (Prometheus + Grafana) with basic system metrics visible.
1.0 Monitoring meaningfully integrated; dashboards clearly reflect system behaviour and are used during deployment or failure demonstration.
5. Mandatory Demonstration (Video Demo and/or Live Presentation) (1.0 point)
Score Description
0.0 Demo missing or heavily staged; CI/CD flow not demonstrated end-to-end.
0.5 Partial demo; some stages shown but missing production verification or monitoring evidence.
1.0 Complete end-to-end demonstration from code change to production update, HTTPS verification, monitoring inspection, and failure simulation.
6. Technical Report (Academic Quality & Evidence Integration) (1.0 point)
Score Description
0.0 Report missing, incomplete, or lacks technical coherence.
0.5 Report complete and understandable but limited in analysis or evidence integration.
1.0 Professional academic report with clear structure, justified design decisions, diagrams, and well-integrated evidence.
Important Submission Notes (Not Graded Separately):
  • Submission completeness and professionalism are mandatory but not scored separately.
  • Any feature or functionality will only be recognized for scoring if it appears in both the video demo (and/or live presentation) and the technical report.
  • Features implemented but not demonstrated and documented will be treated as non-existent for grading purposes.
